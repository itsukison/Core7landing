# Core7 Design System

Core7 is built as a text-native website. The page should feel like a terminal surface, a signal readout, and a product manifesto at the same time. The design language is intentionally narrow: monochrome type, ASCII motion, sparse copy, large empty space, and small interaction details.

## Principles

- Render the brand through text, not illustration.
- Keep the page monochrome: near-black background, off-white foreground, dim gray secondary text.
- Treat whitespace as structure. Sections should feel quiet and deliberate.
- Use motion as signal. Animations should be short, text-based, and mechanical rather than decorative.
- Avoid gradients, glow effects, soft cards, rounded panels, and stock imagery in the core brand surface.

## Typography

The site uses `IBM Plex Mono` as the main interface and rendering font, with Japanese fallbacks for localized copy. The hero wordmark is rasterized from `Space Grotesk` and then translated back into monospace characters.

Key traits:

- uppercase labels
- visible section numbering
- low contrast secondary text through `--dim`
- dense but readable line height
- letter spacing only on small labels, nav items, and command-like links

## ASCII And Halftone Method

The hero graphics are generated by `src/ascii/engine.js`. The engine renders into a full-screen `<pre>` element, so every visual is made from real characters on a fixed monospace grid.

There are two rendering modes:

1. Field scenes

   A shape is drawn to an offscreen canvas first, then sampled cell by cell. Each cell receives a brightness value, and brightness is mapped to characters. This creates the halftone wordmark effect: dense characters in bright areas, sparse punctuation on edges, and empty cells in the background.

2. Band scenes

   Analytic shapes return a left and right edge for each row. The engine fills the row with a character ramp such as:

   ```text
   NNNN0000AAAA88886666999944445555222IIII333???!!!
   ```

   This produces solid ASCII objects with visible horizontal density bands.

The halftone is not a CSS filter or bitmap texture. It comes from three choices working together:

- a measured monospace grid
- brightness sampling per cell
- character density ramps used as tonal values

## Hero Fire Graphics

The fire-like hero treatment is created with the same ASCII halftone logic rather than with an image asset. The technique is:

1. Establish a silhouette.

   The engine starts from a readable graphic mask. For the wordmark, that mask is rasterized text. For abstract hero graphics, the mask can be an analytic interval per row: narrow at the base, widening through the body, then tapering into broken tips.

2. Convert mass into character density.

   Bright or interior areas use dense glyphs such as `!`, `N`, `0`, `A`, `8`, and `6`. Edges use unstable punctuation such as `~`, `"`, `;`, `:`, `*`, and `'`. Outer fade areas use sparse marks such as `.`, `,`, `-`, and `` ` ``.

3. Break the edge.

   A deterministic hash function gives each cell stable noise. This keeps the graphic from looking like a smooth vector shape while avoiding random flicker across the whole canvas. The fire edge should look torn, charred, and alive.

4. Add shimmer selectively.

   The engine reseeds a small share of cells over time. Interior cells stay mostly stable, while edge cells re-roll punctuation. This gives the flame surface motion without making the wordmark unreadable.

5. Use particles as sparks.

   Pointer movement spawns short-lived punctuation particles. They overwrite grid cells, drift, fall, fade to dots, and disappear. This makes the hero respond to the visitor while staying inside the ASCII system.

6. Dissolve between scenes.

   Scene changes are not crossfades. Cells switch from the previous scene to the next through a hash-based dissolve. This preserves the terminal/readout feeling and avoids soft visual blending.

## Character Palettes

Use a small alphabet for each role:

```text
Dense core:    ! N 0 A 8 6 9 4 5 2 I 3 ? 
Hot edge:      ~ " ; : * '
Ash/sparks:    . , - `
Interface:     < > ! ? ; : * = # %
```

Keep new graphics legible from a distance. If the shape disappears when squinting, increase dense characters in the core. If it feels too smooth, increase punctuation at the boundary.

## Interaction

Navigation and landing-page text use a scramble hover effect. The original string is stored in `data-text`, then unsettled characters are temporarily replaced with symbols from the interface palette. Characters settle left to right over a short interval.

Rules for this interaction:

- Keep the original copy unchanged after the animation completes.
- Preserve spaces so line rhythm remains recognizable.
- Use it on text elements, labels, list items, and command-like links.
- Avoid applying it to imagery or large layout containers.
- Keep duration short enough that reading is delayed, not blocked.

## Layout

The landing page is a sequence of quiet text fields:

- fixed text nav
- full-viewport ASCII hero
- centered manifesto sections
- wide product section for the screenshot
- minimal footer columns

Do not introduce card stacks, decorative panels, or marketing hero layouts. The first screen should remain the ASCII experience.
